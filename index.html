  • 크롤링, 데이터 정제, 증강 등의 과정을 포함한 데이터 파이프라인을 설계하고 구현했습니다.
  • 데이터 증강, 랜덤 문장 순서 변환, 랜덤 문장 제거 등의 기법을 적용하여 데이터의 다양성과 품질을 개선했습니다.
  • QA쌍 형태의 구조화된 데이터를 생성하고 DB를 통해 관리했습니다.

    • 핵심 도전 과제, 해결 논의

    과제1. 방대한 크롤링 범위로 인한 핵심 키워드 선정의 어려움

    초기 데이터 수집 과정에서 사용자의 수요를 파악하고, 그에 맞는 키워드를 기반으로 데이터를 수집해야 할 필요성을 느꼈습니다. 저는 학교 커뮤니티의 최근 글들에서 물음표로 끝나는 질문들을 추출해 조사했고, 추가로 설문조사를 통해 키워드를 찾아냈습니다.
    찾아낸 키워드들과 내부 회의를 통해 필수적으로 들어가야 하는 키워드를 최종 결정했고, 이를 통해 학교 커뮤니티, 학사 내규에서 정보를 추출해 데이터를 구축, 증강했습니다.

    과제2. Question-Answer쌍 데이터 생성

    내부 회의를 통해 챗봇의 핵심 기능을 유저 쿼리와 저희 DB의 가장 유사한 질문과의 유사도를 계산해 그에 알맞은 답변을 출력하는 것으로 정했습니다. 이에 따라 Question-Answer 데이터를 생성해야했고 과정은 다음과 같았습니다.

    1. 앞서 수집한 문서를 GPT input에 알맞게 chunk단위로 분할
    2. GPT-turbo-3.5 api를 활용해 앞서 수집된 문서에 대한 Question생성
    3. 생성된 Question + 문서를 입력으로 신뢰성있는 Answer를 생성
    4. 사람이 최종 검수
    각 과정에는 알맞은 GPT 프롬프트를 직접 작성해 효율을 높였습니다.

    과제3. 수집된 데이터의 제한된 다양성

    이론과 달리 실제 사용자는 기대했던 문장과 다른 방식으로 질문을 하는 경우가 많습니다. 예를 들어, 학교 이름을 줄이거나, 과목 이름을 줄이는 방식이 있겠습니다. 저희는 이를 감안해 임의로 유사어/준말에 대해 질문 데이터를 증강했습니다.
    또 한국어는 어순이 달라져도 의미가 유지되는 경우가 많습니다. 저희는 이를 감안해 문장 내 단어를 랜덤으로 스왑해 데이터를 증강했습니다.
    최종 증강된 데이터는 수집한 QA에 대해서 약 4만개 입니다.

    1. GPT 프롬프트를 통해 질문 포맷을 지정해 다양한 질문 유형 생성
    2. 유사어/준말 어휘에 대한 증강
    3. 랜덤 문장 순서 변환 및 제거를 통한 추가 데이터 다양화

    2. Deploying Model

    주요 업무
  • AWS EC2를 활용한 AI 모델 배포.
  • Flask 기반 API 개발로 모델 배포 및 데이터 파이프라인 구축.
  • MySQL을 사용한 관계형 데이터베이스 설계 및 운용.

    • 핵심 도전 과제, 해결 논의

    과제1. AWS EC2 인스턴스 선정

    EC2 인스턴스 비용에 알맞은 모델을 선정하는 과정은 까다로웠습니다. 저희에게 할당된 비용은 한정되어 있고, 사용하고자 하는 모델은 거대했습니다. 저희는 되도록 가벼운 모델을 사용하고, 양자화(LoRA)를 통해 모델 크기를 줄였고, EC2 인스턴스 유형으로 t2.medium을 사용해 최대한 메모리를 늘리고자 했습니다.

    과제2. Flask를 통한 api 배포

    LLM을 배포하고 백엔드와 연동하는 작업은 당시 정보가 많이 없었기에 어려운 작업이었습니다. 배포과정에서 주요 고려사항은 모델의 추론 시간과 출력 시각화 방식이었습니다. 저희는 추론 시간을 저희는 앞서 서버에 올린 모델을 Flask를 통해 api로 제공하고, Gradio를 통해 모델 출력을 스트리밍 방식으로 시각화해 Client에게 제공했습니다. 주요 프로세스는 다음과 같습니다.

    1. AWS에서 EC2 인스턴스를 생성하고 필요한 환경을 설정
    2. 모델을 EC2 인스턴스에 업로드
    3. Flask를 이용해 모델을 API로 제공
    4. 사용자 상호작용을 위한 Gradio 인터페이스 활용
    5. Gradio 인터페이스와 Flask API를 연결하고 시스템 테스트

    ForeignAICC

    Main Poster

    The project was started to participate in a generative AI competition for solving social problems organized by the Seoul Metropolitan Government. The project is currently in the main round after passing the preliminary round.

    Key features

    1. Foreign workers consult with a chatbot about what they want to know (legal, visa questions, etc.).
    2. The chatbot provides legal information, examples, and specific solutions based on data.
    3. If further counseling is needed, it is possible to schedule a consultation directly from within the app.
    4. To reduce the effort of offline consultations, we provide a chat summary to the counselor to help make the process smoother.

  • prefix + documents + query 합쳐서 LLM에 input으로 입력

    • 핵심 도전 과제, 해결 논의

    과제1. 다국어 처리 방법

    초기에 사용자들의 다양한 외국어 입력을 어떻게 처리해야 할지 고민이었습니다. 다국어 모델을 사용하는 것을 고려해보았지만, 결국 성능을 위해 따져 영어 모델을 중심으로 입/출력을 GPT api를 통해 번역하는 방식으로 구현하게 되었습니다. 저희는 성능이 입증된 mistral-7b/llama3b 모델을 최종 선정했습니다.

    과제2. RAG(Retrieval-Augmented Generation) 활용

    저희는 RAG 기법 활용을 위해 현재 LangChain에 기반한 elastic search로 유사도를 계산하고 K개의 유사한 문서를 참고하는 방법을 계획했습니다.

    과제3. STT/TTS 유선 전화 연결

    현재 Twillio를 활용해 구현 중입니다.
